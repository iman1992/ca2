REST API

We have our "CRUD" operations in the "RestService" class - getAllPersons, getPerson, createPerson, delete and edit. The "RestService" has access to the "Facade" class, which has a connection to the database. The RESTful API is just a simple collection of URIs, HTTP calls to those URIs and some JSON representations of resoursces, many of which contain relational links. Each resouse has its own address or URI - every thing the server can provide is exposed as a resource.

To insert (create) a new person in the system, we use:

POST - http://localhost:8080/CA2NewVersion/api/rest/add

To read all persons in the system, we use:

GET - http://localhost:8080/CA2NewVersion/api/rest

To read a person with id = 1, we use:

GET - http://localhost:8080/CA2NewVersion/api/rest/1

To delete a person with id = 1, we use:

DELETE - http://localhost:8080/CA2NewVersion/api/rest/delete/1

To edit a person with id = 1, we use:

PUT - http://localhost:8080/CA2NewVersion/api/rest/edit/1

Facade Design Pattern

We have implemented the Facade Desing pattern in our system. The facade provides a unified interface to a set of interfaces in a subsystem. The facade makes the subsystem easier to use, because of the defined higher-level interface. In our application the Facade has a connection with the MySQL database.

JSON Format

JSON (JavaScript Object Notation) is a data-interchange format. It is easy to read and write. It is easy for machines to parse and generate. It is based on a subset of the Javascript Programming Language. JSON is a collection of name/value pairs.
